# Product Overview

Crownfall compresses a fantasy campaign arc into a 35–45 minute social-deduction match for 6–10 players. A rotating Regent nominates a Pathfinder, the council votes, a party commits hidden Quest Sigils, and public evidence accumulates toward Hope, Dread, Ruin, or a scenario ending. Banished players remain involved as Fallen players.

The server is the impartial Storyteller. It validates every action, protects hidden roles and objectives, and produces a deterministic Chronicle from the append-only match record. The browser provides accessible React flows and a PixiJS animated table. Voice is product-critical but belongs to a separate WebRTC/SFU plane and is not part of this foundation.

The complete rules, roles, scenarios, balance targets, UI flows, and acceptance criteria are normative in the [GDD](../crownfallGameDesignDocument.md).
